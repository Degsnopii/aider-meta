### ❌ ОШИБКА #1: При первом запуске сам пишет название "пингер"
**Тип:** Логическая ошибка при инициализации конфига  
**Строки кода:** 230 (дефолт PROJECT_NAME)  
**Проблема:**  
```bash
: "${PROJECT_NAME:=$(basename "$PWD")}"  # строка 230
```
Если скрипт запущен из папки с "неудачным" именем, оно берётся как дефолт.

**Решение:**  
- Добавить интерактивный prompt при первом запуске  
- Запросить имя проекта в `first_run_wizard` ДО load_config  
- Сохранить в конфиг  

**Статус:** 🔴 **ТРЕБУЕТСЯ ИСПРАВЛЕНИЕ**

### ❌ ОШИБКА #2: Не переключается на версию текст консоль меню
**Тип:** UI режим не срабатывает  
**Строки кода:** 657-663 (theme_settings_tui), 232 (UI_MODE переменная)  
**Проблема:**  
- Переключение режима в `UI_MODE` сохраняется  
- Но главное меню ВСЕГДА использует `whiptail`, никогда не проверяет `UI_MODE`

**Решение:**  
- Создать функцию `main_menu_console()` для текстового режима  
- В `main()` проверить `$UI_MODE` и вызвать нужное меню  
- Добавить логику переключения для всех интерфейсов  

**Статус:** 🔴 **ТРЕБУЕТСЯ ИСПРАВЛЕНИЕ**

### ❌ ОШИБКА #8: Переход из TUI whiptail в Console text должен быть ТОЛЬКО при запуске Aider
**Тип:** Логическая ошибка (MAJOR)  
**Строки кода:** 127-145 (run_logged_command), 1251 (launch_aider)  
**Проблема:**  
- Сейчас весь лог выводится в отдельном textbox (tailbox)  
- При скачивании модели, запуске Ollama, установке зависимостей — выскакивает отдельное окно  
- Это нарушает flow пользователя  
- **Должно быть:** логи ВНУТРИ whiptail окна, обновляться в реальном времени  
- **Исключение:** ТОЛЬКО при прямом запуске Aider или переключении режима UI  

**Решение:**  
- Создать интерактивный `whiptail_log_viewer()` который:
  - Показывает логи в реальном времени (tail -f в подпроцессе)
  - Выводит в infobox/textbox БЕЗ окончания диалога
  - Обновляет каждые 0.5 сек
- Применить ко ВСЕМ операциям: СКАЧАТЬ МОДЕЛЬ, УСТАНОВИТЬ, ЗАПУСТИТЬ OLLAMA, ТД...
- Исключение: при `launch_aider` переходить в консоль нормально (exec)

**Статус:** 🔴 **КРИТИЧЕСКАЯ ОШИБКА x10** — влияет на весь UX
